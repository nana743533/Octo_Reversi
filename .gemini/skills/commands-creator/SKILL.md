# Commands Creator (コマンドクリエイター)

このスキルは、プロジェクトの効率を高めるためのカスタムコマンド（自動化スクリプト）を作成・管理する手順を提供します。

## コマンドとは

カスタムコマンドは、`.gemini/commands/` に配置された実行可能なスクリプトです。Gemini CLI を通じて、または直接シェルから実行できるプロジェクト固有の道具箱となります。

## 作成ワークフロー

1. **ニーズの特定**: 繰り返されている手動操作や、複雑な手順を特定します。
2. **言語の選択**: シェルスクリプト (Bash)、Python、Node.js など、タスクに最適な言語を選びます。
3. **実装**: `.gemini/commands/` 配下にスクリプトを作成します。
4. **権限付与**: `chmod +x` で実行権限を付与します。
5. **ドキュメント更新**: `.gemini/commands/README.md` に新しいコマンドの説明を追記します。

## 詳細ガイド

具体的な作成手順やベストプラクティスについては、[references/guidelines.md](references/guidelines.md) を参照してください。

## コマンドの例

- `setup-db.sh`: 開発環境のデータベースを初期化し、シードデータを投入する。
- `gen-api-client.py`: OpenAPI定義からクライアントコードを生成する。
- `check-health.sh`: ローカルサーバーのヘルスチェックを行う。

## 注意事項

- 機密情報（APIキーなど）をスクリプト内にハードコードしないでください。環境変数を使用してください。
- 破壊的な操作を行うコマンドは、実行前に確認プロンプトを出すか、十分な警告を表示してください。
