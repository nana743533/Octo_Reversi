# /commit

このコマンドは、Semantic Commit Message 規約に従い、日本語でコミットメッセージを作成してコミットを行います。

## 手順

1.  **ステージング済みの変更を確認**: `git diff --cached --name-only` を実行して、現在ステージングされているファイルを確認します。
    - **重要**: ステージングされている変更（`git add` 済みのもの）のみをコミット対象とします。未ステージングの変更は無視してください。
    - もしステージングされているファイルがない場合は、ユーザーに「ステージングされている変更がありません」と伝えて処理を終了します。
2.  **コミットメッセージの作成**: ステージングされている変更内容（`git diff --cached` で確認）に基づいて、以下の Semantic Commit 規約に従い、**日本語**でメッセージを作成します：
    -   形式: `<type>(<scope>): <description>` （scopeは任意）
    -   type: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`, `build`, `ci`, `revert`
    -   description: 日本語で簡潔に記述。
3.  **コミット**: `git commit -m "<message>"` を実行します。
4.  **プッシュ**: コミット完了後、`git push` を実行してリモートに反映します。

## 制約
- メッセージは必ず日本語で書くこと。
- Semantic Commit Message の形式を厳守すること。
- 複数の関心事が混ざっている場合は、適切にコミットを分けること。
