# Commands

このディレクトリには、プロジェクト固有のカスタムコマンド定義を配置します。
よく使う一連の操作をコマンドとして定義しておくことで、効率的に開発を進めることができます。

カスタムコマンドは **TOML形式** (`.toml`) で定義する必要があります。

## 構成
各 `.toml` ファイルは以下のキーを持ちます：
- `description`: コマンドの説明（`/help` で表示されます）。
- `prompt`: AIモデルに送信されるプロンプト。
    - `{{args}}`: ユーザーが入力した引数に置き換わります。
    - `!{command}`: シェルコマンドの実行結果に置き換わります。
    - `@{file}`: ファイルの内容に置き換わります。

## 例
- `setup-env.sh`: 環境構築スクリプト（シェルスクリプトとして直接実行）
- `deploy.sh`: デプロイスクリプト
- `commit.toml`: Semantic Commit Message（日本語）による対話的なコミット

### commit.toml の例
```toml
description = "コミットを自動化します"
prompt = """
以下の変更をコミットしてください。

変更内容:
!{git diff --cached}

メッセージは日本語で作成してください。
"""
```
